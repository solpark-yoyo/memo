## Look-up-ahead Proxy 설계 결과 (260621, multi-agent: 6관점 독립설계 → 적대비평 → 통합)
접근: look-up-ahead proxy를 6개 수학적 관점에서 독립 설계하고, 각각 적대적으로 비평(judge)한 뒤 하나로 통합.

[근본 발견 — early step의 본질적 한계]
측정된 SD1.5 schedule 기준, update step에서 alpha_t 가 매우 작다:
  - t_idx=10: alpha_t = 0.0435, SNR = 0.045, Tweedie amplification 1/sqrt(alpha) = 4.80
  - t_idx=19: alpha_t ≈ 0.16
즉 x0_hat = (z_t - sqrt(1-alpha_t) eps_theta)/sqrt(alpha_t) 는 eps_theta 오차를 2.5~4.8배 증폭 → noise-scaled garbage.
그래서 x_s = sqrt(alpha_s) x0_hat + sqrt(1-alpha_s) x_T 가 모든 horizon s 에서 x_T 항에 지배됨.
→ eps_theta(x_s, s) 가 memorized/normal 모두 near-pure-noise 입력에 대한 동일 smooth 응답 → gap 에 prompt-판별 신호 없음.
→ baseline(||eps_ref - eps_s||^2) 포함 모든 x0_hat-의존 proxy 가 early 에서 blind.
   (이게 detach 수정 후에도 metric 이 DDIM 과 비슷할 수 있는 구조적 이유이기도 함.)

[6개 설계 평가 (score / verdict)]
  [4.0 weak]  CURC — CFG residual r_t = eps_c - eps_uc 기반. x0_hat 독립이라 early 에서 유일하게 살아남음.
  [3.5 weak]  MTTG — multi-horizon telescoping. early 에서 모든 horizon 이 같은 x_T 를 보는 문제.
  [3.5 weak]  CSRT — round-trip self-consistency. early 에서 near-identity(trivial) 또는 late(time-mismatch 재발).
  [3.5 weak]  look-ahead — k-step forward predictor. reference-identity mismatch (gap to x_T != gap to x0_mem).
  [2.5 reject] Lyapunov — trajectory divergence. early score 가 prior-dominated 라 Jacobian near-identity + fp16 quantization floor.
  [2.0 reject] Fisher/curvature — JVP differentiability 가 거짓 (codebase JEPA code 가 finite-diff Jv no-grad 로 문서화).

[살아남은 신호: CRELA (CFG-Residual Early-Look-Ahead)]
유일하게 early 에서 informative 인 것은 CFG residual r = eps_c - eps_uc.
  - 두 UNet branch 차이 → 큰 shared denoising direction 이 cancel → conditioning pull 만 남아 SNR 높음.
  - x0_hat 필요 없음 → early step blind 회피.
  - baseline 이 이미 계산하는 branch 재사용 → 추가 UNet call 0.
정의: rho_t(x) = mean(r(x,t)^2) / (mean(eps_uc(x,t)^2) + eps_floor)   (relative condition pull)
  - early term:    M_E    = rho_t(z_t)                 (grad: z_t -> x_T)
  - late look-ahead (s_idx=25, SNR≈0.43): rho_s = rho_t'(x_s)   (grad: x0_hat, x_T -> x_s)
  - rho_change = rho_s - M_E.detach()
  - loss = sign_E * M_E + sign_L * rho_s + 0.3 * rho_change^2 + lambda_align * ||x0_hat - x0_orig||^2

[핵심 주의: sign ambiguity]
memorized 에서 rho 가 작아지는지(CURC claim: condition-free 재현) vs 커지는지(prompt-conditioned overfit)는 기계적으로 미정.
→ 5-line diagnostic 이 최우선: memorized vs normal prompt 에서 rho_t, cos(eps_c, eps_uc) 측정 → sign_E, sign_L 결정.
이 diagnostic 가 sign 를 못 정하면 CRELA 전체 부호가 뒤집힐 수 있음 (비평가가 지적한 CURC killer flaw).

[다음 실험 우선순위 (open questions)]
1. gating diagnostic: memorized/normal prompt × t_idx∈{10,13,16,19} 에서 rho_t, cos(eps_c, eps_uc) 측정 → sign 결정. 모든 것보다 먼저.
2. rho_t 가 SNR 변화(0.045→0.19)에 안정적인지, 아니면 per-step normalize 가 필요한지.
3. x_T 를 rho de-memorizing 방향으로 optimize 했을 때 downstream memorization metric(MSS / pixel similarity) 이 실제 감소하는지 (end-to-end ablation). optimizer 가 rho 만 spoof 하고 진짜 de-memo 를 안 할 수 있음.
4. late look-ahead rho_s 가 s_idx=25 에서 attractor 를 반영하는지, 더 late(s>30)로 가야 하는지.
5. fp16 안정성: ratio rho 가 overflow 가능, eps_floor=1e-3 충분한지 (또는 ratio 만 fp32 upcast).

[fallback] CRELA-lite: L = sign_E * M_E + lambda_align * ||x0_hat - x0_orig||^2  (look-ahead/consistency 항 제거, x0_hat 독립 early term 만). 추가 compute 0.

=> 내가 느끼는 문제점은 Tweedie formula의 SNR을 해결이다. 
사실 tweedie formula를 정리하면 x_0 + (계수)* (noise)의 형태를 나타낸다. 이때 SNR이 너무 낮아버리면 x_0|t에 conditional 정보가 거의 사라진다고 볼 수 있다.
이때 noise앞에 계수는 alpha_t가 클수록 계수가 낮아진다. 사실 당연한 직관이다. alpha_t(latent에서 clean image의 비중)이 높을 수록 
그 시점의 latent는 clean image를 많이 포함하고 있다는 것이고 이는 signal을 많이 포함하는 x_0|t를 만들어낼 수 있다. 즉 SNR이 높다.
그래서 나는 지금 Tweedie formula를 통과할 때마다 alpha_t, SNR, Tweedie formula의 계수 까지 출력하는 코드를 추가했다. 근데 내생각에는ㄴ SNR을 높히려고

--- 문제점 & 해결 (260622) ---

[문제 1] x_s 재구성 시 x_T shortcut gradient
  - x_s = √(α_s)·x̂₀ + √(1-α_s)·x_T 에서 x_T가 grad-tracked → x_T→x_s 직접 shortcut 발생
  - 의도한 gradient: x_T → denoising chain → x̂₀ → x_s → ε_s → loss (trajectory 경로)
  - 실제: x_T → x_s 직접 shortcut이 trajectory 경로를 무시하고 지배 → 비정상적 x_T 이동 → 이미지 부서짐
  - 해결: x_s의 x_T를 .detach() → shortcut 제거, gradient가 trajectory 경로로만 흐름 (eps_trajectory.py:107 참조 구조와 정합)

[문제 2] loss_align gradient vanishing → lambda_align flat
  - loss_align = ||x̂₀(x_T) - x̂₀_orig||² 의 gradient가 10~19 UNet steps (fp16)를 거쳐 x_T에 도달해야 함
  - 이 긴 chain에서 gradient 소실 → lambda_align이 x_T를 실제로 제어 못 함
  - loss_memo는 (문제1 수정 전엔) x_s shortcut으로 direct path가 있어 강했음 → 비대칭
  - 해결 방향: loss_align을 x_T 직접 anchor(||x_T - x_T_init||²)로 변경하면 direct gradient 확보 (TODO)

결국에 문제는 지금까지 init_opti의 grad는 x_s에서 재활용되었던 x_T에 흘러가는 grad였다. 하지만 이는 틀렸다.
  - 의도한 gradient: x_T → denoising chain → x̂₀ → x_s → ε_s → loss (trajectory 경로)
의도한 경로로 grad를 흐를 수 있도록 수정을 했으나 gradient vanishing 문제가 발생했다. 그래서 지금은 denoising step을 많이 가져간 x_t에서도
backprop이 정상적으로 걸릴 수 있을지 시도를 해야 된다.

어떤 방법을 사용해야 하는지 난감하다.

현재 시점에서 가장 중요한 문제: initial noise까지 loss의 grad가 흘러가지 않는다.
그 이유는 DDIM 에서 denoising step에 사용되는 UNet이 많다보니, gradient vanishing 문제가 발생한다.
그 문제를 해결하기 위해서 adjointDPM이라는 방법을 적용해야겠다.

# 260626 (Fri): Tweedie_opti 구현

문제는 DDIM의 long sequential UNet Chain으로 grad를 흘리는 과정에서 생길 vanishing 문제였고 이를 adjointDPM 이라는 방법을 통해서 위 문제를 해결했다.
adjointDPM의 주요 핵심은 latent까지 흘러가는 grad를 곱셈이 아닌 덧셈 과정으로 처리한 것이다.
원래의 방식은 loss의 computational trajectory에 있는 모든 과정의 grad를 곱셈으로 계산을 했다.
하지만 위 방식은 x_t에 흘려줄 grad를 time t 의 adjoint state라고 정의하고
adjoint state의 differention의 formulation을 함으로서 grad 계산을 덧셈으로 바꾼 것에 있다.
위 문제가 OOM과 gradient vanishing을 해결하리라 예상했고 구현에 성공했다.
